# 跨插件 JPA 事务能力实施计划（追踪版）

## 1. 目标与范围

- 按「可选能力」实现跨插件同域事务。
- 保留非 JPA 插件零影响。
- 使用现有 `pf4boot-jpa` / `pf4boot-jpa-starter` 扩展为 `LOCAL/SHARED` 模式。
- 本计划仅覆盖第一阶段范围：同域共享事务，多域隔离，不支持跨域原子事务。

## 4. 关键依赖与前置条件

- `pf4boot-jpa`、`pf4boot-jpa-starter` 现有代码路径清晰、可在兼容基础上改造。
- 根仓库已完成依赖树梳理，且插件打包链路可复用新模块。
- 验收环境可运行 `gradlew`（至少 `compileJava` 与 sample 插件打包）。
- 测试环境支持可控地引导能力插件启动失败场景（用于隔离验证）。

## 5. 风险缓冲与延期策略

- 若 `M2` 遇到兼容性问题，优先回退：不改动 `pf4boot-jpa-starter` 默认行为；先在非共用路径实现。
- 若样例启动链路不稳定，暂停 `M4` 并回归最小编译链。
- 采用两轮交付策略：先 `LOCAL` 不变 + `SHARED` 可选；再补齐完整样例验证。
