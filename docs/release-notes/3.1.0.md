# pf4boot 3.1.0 Release Notes

## 发布定位

`3.1.0` 是一次插件管理边界收敛版本，重点将基础插件管理与 JPA reload 管理能力解耦，便于宿主应用只引入插件 list/start/stop/restart/reload/enable/disable 与 plugin-deploy 能力。

## 主要变化

- 新增 `pf4boot-jpa-management-starter` 可选模块，用于显式启用 JPA reload HTTP 接口和 `pf4bootjpareload` Actuator 端点。
- `pf4boot-management-starter` 不再强依赖 `pf4boot-jpa`，基础管理 starter 只提供插件基础管理和部署能力。
- `pf4boot-actuator` 不再强依赖 `pf4boot-jpa`，JPA reload 观测端点随可选 JPA 管理 starter 启用。
- 移除 `pf4boot-web-starter` 中旧的 `PluginManagerController` 管理接口，只保留 `pf4boot-management-starter` 提供的 `/pf4boot/admin/**` 新插件管理接口。
- 插件管理 HTTP 合同收敛为基础接口，CLI 默认不应生成 `plugin-jpa-reload` 命令。
- `samples/cross-plugin-jpa` runtime smoke 不再写死插件 zip 版本号，改为从 runtime 插件目录动态解析当前构建产物。

## 发布范围

发布到 Maven 仓库的模块不包含 `samples/*` 项目。当前 publishable 模块为：

- `pf4boot-api`
- `pf4boot-core`
- `pf4boot-starter`
- `pf4boot-web-support`
- `pf4boot-web-starter`
- `pf4boot-jpa`
- `pf4boot-jpa-starter`
- `pf4boot-jpa-domain-starter`
- `pf4boot-jpa-management-starter`
- `pf4boot-actuator`
- `pf4boot-management-starter`

## 兼容性说明

- 继续保持 Java 8 兼容。
- 旧 `PluginManagerController` 管理接口已移除，管理客户端应切换到 `/pf4boot/admin/**` 新接口。
- 只需要基础插件管理的宿主应用引入 `pf4boot-management-starter` 即可，不会被动引入 `pf4boot-jpa`。
- 需要 JPA reload 管理能力的宿主应用必须额外引入 `pf4boot-jpa-management-starter`。
- sample 项目只用于演示和验收，不作为 Maven 发布物。

## 发布前验证

本版本发布前建议执行：

```powershell
.\gradlew.bat build
.\gradlew.bat :pf4boot-management-starter:test :pf4boot-actuator:test :pf4boot-jpa-management-starter:test --rerun-tasks
.\gradlew.bat :pf4boot-management-starter:dependencies --configuration runtimeClasspath
.\gradlew.bat :pf4boot-actuator:dependencies --configuration runtimeClasspath
.\gradlew.bat publishToMavenLocal
```

如需回归 sample 行为，可额外执行：

```powershell
.\gradlew.bat :samples:cross-plugin-jpa:app-run:runtimeSmoke
```

## 发布后建议

- 使用 `v3.1.0` tag 对应的源码生成正式构建。
- 发布页面引用本文档作为 release notes。
- 下一轮开发版本建议切到 `3.1.1-SNAPSHOT`。
