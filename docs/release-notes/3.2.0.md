# pf4boot 3.2.0 Release Notes

## 发布定位

`3.2.0` 是一次生产化闭环版本，重点把 JPA 运行时刷新、热替换部署事务、HTTP 管理治理、资源清理诊断和 runtime smoke 验收收敛为可发布能力。

## 主要变化

- 新增下一版本生产化目标和设计文档，明确 G1-G5 的范围、验收和兼容边界。
- JPA reload 管理接口新增独立权限点：`JPA_RELOAD_PLAN`、`JPA_RELOAD_EXECUTE`、`JPA_RELOAD_QUERY`。
- JPA reload 执行接口接入写请求安全校验、幂等、operation record、审计和失败记录。
- 新增 `PluginCleanupSummary`，部署记录和 JPA reload 记录均可暴露资源清理摘要。
- `PluginDeploymentService` 新增兼容 default 方法：`getRecord`、`rollback(String)`、`rollback(DeploymentRecord)`。
- `DefaultPluginDeploymentService` 支持部署记录查询和显式 rollback，并在热替换成功路径记录 cleanup summary。
- JPA provider replacement summary 新增 `errorCode`，并保留旧构造器和旧记录兼容。
- JPA reload 文件记录仓库增强测试覆盖 provider replacement summary、cleanup summary、latest 和 idempotency 恢复。
- `samples/cross-plugin-jpa` 的 Gradle `runtimeSmoke` 覆盖部署失败预检、JPA reload、provider replacement、drain timeout、record persistence、Actuator 汇总和无关插件隔离。
- `samples/cross-plugin-jpa:app-run:assembleSampleRuntime` 每次清理完整 runtime 输出目录，避免旧版本 jar 污染 `lib/*` 类路径。

## 发布范围

发布到 Maven 仓库的模块不包含 `samples/*` 项目。当前 publishable 模块为：

- `pf4boot-api`
- `pf4boot-core`
- `pf4boot-starter`
- `pf4boot-web-support`
- `pf4boot-web-starter`
- `pf4boot-jpa`
- `pf4boot-jpa-starter`
- `pf4boot-jpa-domain-starter`
- `pf4boot-jpa-management-starter`
- `pf4boot-actuator`
- `pf4boot-management-starter`

## 兼容性说明

- 继续保持 Java 8 兼容。
- 新增 JPA reload 管理权限会影响自定义 `PluginManagementAuthorizer`；默认本地 token authorizer 已放行。
- `PluginDeploymentService` 新增方法均为 default method，不破坏二进制兼容。
- `DeploymentRecord`、`JpaDomainReloadRecord`、`JpaProviderReplacementSummary` 新增字段均保持旧构造器兼容；旧 JSON 记录缺失新增字段时按 `null` 处理。
- `PluginCleanupSummary` 是新增只读模型，管理响应新增字段保持 JSON 向后兼容，旧客户端可以忽略。
- JPA reload 默认仍保持受配置控制；未引入 `pf4boot-jpa-management-starter` 时不会注册 JPA 管理入口。
- sample 项目只用于演示和验收，不作为 Maven 发布物。

## 发布前验证

本版本发布前建议执行：

```powershell
.\gradlew.bat build
.\gradlew.bat :pf4boot-core:test :pf4boot-management-starter:test :pf4boot-jpa-starter:test :pf4boot-jpa-management-starter:test --rerun-tasks
.\gradlew.bat :samples:cross-plugin-jpa:app-run:runtimeSmoke
.\gradlew.bat publishToMavenLocal
```

## 已执行验证

## 发布后建议

- 使用 `v3.2.0` tag 对应的源码生成正式构建。
- 发布页面引用本文档作为 release notes。
- 下一轮开发版本建议切到 `3.2.1-SNAPSHOT` 或 `3.3.0-SNAPSHOT`，取决于是否继续新增生产化能力。
