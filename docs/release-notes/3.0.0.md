# pf4boot 3.0.0 Release Notes

## 发布定位

`3.0.0` 是一次面向生产治理能力的主版本发布，重点增强插件部署、HTTP 管理接口、JPA 运行时刷新、Actuator 观测和复杂 sample 验证。

## 主要新增能力

- 新增 `pf4boot-management-starter`，提供插件查询、生命周期、部署预检、替换、确认、回滚和 JPA reload HTTP 管理接口。
- 新增管理接口鉴权、幂等、限流、CSRF/Origin 校验、审计记录和统一错误码。
- 新增插件部署记录与 operation store，支持内存和文件型记录存储。
- 新增 JPA domain reload 计划、执行、记录查询、drain 协调和 provider replacement path。
- 新增 `pf4boot-actuator` 管理治理、部署和 JPA reload 观测指标。
- 新增 offline-index 插件仓库解析、release request、checksum/trust manifest 和版本范围预检相关模型。
- 新增 `samples/cross-plugin-jpa`，覆盖跨插件 JPA domain、consumer、workflow、管理接口和 runtime smoke。
- 新增 `samples/saga-outbox`，展示 Saga/Outbox 事务补偿示例。
- 新增 `samples/plugin-management-console`，作为管理 HTTP API 的前端对接示例。

## 发布范围

发布到 Maven 仓库的模块不包含 `samples/*` 项目。当前 publishable 模块为：

- `pf4boot-api`
- `pf4boot-core`
- `pf4boot-starter`
- `pf4boot-web-support`
- `pf4boot-web-starter`
- `pf4boot-jpa`
- `pf4boot-jpa-starter`
- `pf4boot-jpa-domain-starter`
- `pf4boot-actuator`
- `pf4boot-management-starter`

## 兼容性说明

- 继续保持 Java 8 兼容。
- 管理 HTTP 接口默认关闭，必须显式配置 `spring.pf4boot.management.http.enabled=true` 并选择非 `DISABLED` 模式。
- JPA reload 执行默认受治理配置控制，不会在未启用执行模式时产生运行时变更。
- sample 项目只用于演示和验收，不作为 Maven 发布物。

## 发布前验证

本版本发布前建议执行：

```powershell
.\gradlew.bat build
.\gradlew.bat :pf4boot-core:test :pf4boot-jpa-starter:test :pf4boot-jpa-domain-starter:test :pf4boot-management-starter:test :pf4boot-actuator:test --rerun-tasks
.\gradlew.bat :samples:plugin-management-console:test --rerun-tasks
.\gradlew.bat :samples:cross-plugin-jpa:app-run:runtimeSmoke
.\gradlew.bat :samples:saga-outbox:app-run:runtimeSmoke
.\gradlew.bat publishToMavenLocal
```

## 发布后建议

- 使用 `v3.0.0` tag 对应的源码生成正式构建。
- 发布页面引用本文档作为 release notes。
- 下一轮开发版本建议切到 `3.0.1-SNAPSHOT` 或 `3.1.0-SNAPSHOT`，取决于后续是否继续引入新能力。
